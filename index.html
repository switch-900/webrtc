Manual WebRTC Chat + Pong

Same page for both peers. No signaling server. You manually copy/paste the connection blob between browsers. Once connected, chat and a tiny two-player Pong demo open over a WebRTC data channel.

1) Connection

signaling: idle peer: disconnected data channel: closed role: none
The app waits for ICE gathering to finish, then outputs a single copy/paste blob. That means you usually only need to exchange one blob each way.
How to use:
  1. Peer A clicks Create Offer.
  2. Peer A sends the generated blob to Peer B.
  3. Peer B pastes it into Remote blob and clicks Accept Offer / Create Answer.
  4. Peer B sends their generated answer blob back to Peer A.
  5. Peer A pastes that answer into Remote blob and clicks Accept Answer.
